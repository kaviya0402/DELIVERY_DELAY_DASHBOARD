**📌Objective**

This project focuses on analyzing delivery performance and identifying delay patterns using customer, product, and order data. The dashboard provides insights into on-time delivery performance, delayed orders, and average delivery duration across categories, customer segments and years.

**🛠 Tools Used**

- Microsoft Excel
- Power Query (Data Cleaning & Transformation)
- Pivot Tables
- Pivot Charts
- Data Modeling
  
**📂 Data Model**
  
  The project uses three related tables:

1️⃣  Customer Table
- Customer ID
- Payment Type
- Profit per Order
- Sales per Customer
- Category ID
- Category Name
- Customer City
- Customer Country
- Customer Segment
- Customer State
- Department ID
- Department Name
- Latitude
- Longitude
- Market

**2️⃣  Product Table**
- Product ID
- Product Category ID
- Product Name
- Product Price
- Shipping Date
- Shipping Mode
- Label

**3️⃣  Order Table**
- Order Date
- Order Status
- Expected Delivery Date
- Actual Delivery Date
- Delivery Delay Days
- Delivery Status
  
  Relationships were created between the three tables to enable proper analysis.

**📊 KPIs Created**
- Total Orders
- Delivered Orders
- Average Delivery Days
- On-Time Delivery

**📈 Visualizations**
- Orders by Delivery Status
- Average Delivery Days by Category
- Sales by Product Category
- Delayed Orders by Order Date
- Average Delivery Days by Customer Segment
- Year-wise Delivery Performance

**🎛 Slicers Added**
- Order Date (Year-wise)
- Customer Segment
- Delivery Status
  Slicers allow dynamic filtering of the dashboard for better analysis.

**📌 Key Insights**
- Identified delay trends over time
- Measured on-time vs delayed order performance
- Analyzed delivery efficiency across customer segments
- Compared delivery duration by product
  
**✅ Conclusion**
The Delivery Delay Dashboard provides clear visibility into logistics performance and helps identify operational inefficiencies. The analysis supports data-driven decisions to improve delivery timelines and overall customer satisfaction.
